üìù Why Info Snap!

Info Snap is an intelligent document summarization tool that I built to solve a real problem - we all deal with lengthy PDFs and documents daily, and extracting key information quickly is crucial. This project combines my passion for AI/ML with practical web development to create something truly useful.

üîß Technical Architecture:

The application leverages Facebook's BART (Bidirectional and Auto-Regressive Transformers) model for generating high-quality summaries. BART excels at understanding context and producing coherent, human-like text summaries. For text extraction, I integrated Tesseract OCR which handles image-based documents with impressive accuracy.

üõ†Ô∏è Core Technologies & Libraries:

  • Backend Framework: Flask (Python) - chosen for its simplicity and flexibility in handling AI models
  • NLP Model: facebook/bart-large-cnn - fine-tuned for summarization tasks
  • Text Extraction: PyPDF2 for PDFs, Tesseract OCR for images (PNG, JPG, etc.)
  • Translation API: Google Translate API integration for 15+ language support
  • PDF Generation: ReportLab for creating downloadable summary PDFs
  • Frontend: Vanilla JavaScript with modern ES6+ features, no heavy frameworks needed
  • Styling: Custom CSS with glassmorphism effects and smooth animations
  • File Handling: Werkzeug for secure file uploads with validation

üí° Key Features I Implemented:

  • Smart Text Processing: The app intelligently handles different document types - PDFs are parsed page by page, while images go through OCR preprocessing for better accuracy
  • Adjustable Summary Length: Users can choose between short (3-5 sentences), medium (5-8 sentences), or long (10+ sentences) summaries based on their needs
  • Key Points Extraction: Beyond just summarizing, the app identifies and lists the most important points from the document
  • Real-time Translation: Integrated Google Translate API to break language barriers - summaries can be instantly translated to Hindi, Spanish, French, and 12 other languages
  • Share Functionality: Created a unique link-sharing system where summaries are temporarily stored (24-hour expiry) with view tracking
  • Dark Mode: Because who doesn't love dark mode? Implemented with CSS variables and localStorage for preference persistence

üöÄ Performance Optimizations:

  • File Size Limits: Set 16MB limit to ensure quick processing and prevent server overload
  • Rate Limiting: Implemented custom rate limiting to prevent API abuse (5 requests/minute for uploads)
  • Automatic Cleanup: Files are deleted immediately after processing to maintain privacy and save storage
  • Lazy Loading: Results section only renders after successful summary generation
  • Client-side Validation: File type and size checks happen before upload to save bandwidth

üîí Security Measures:

  • Secure Filenames: Using werkzeug's secure_filename to prevent path traversal attacks
  • CORS Headers: Properly configured to prevent unauthorized API access
  • Input Sanitization: All user inputs are validated and sanitized before processing
  • Temporary Storage: No permanent storage of user documents - privacy first approach

üéØ Challenges I Overcame:

  • OCR Accuracy: Initially, OCR was giving poor results for handwritten text. I solved this by implementing image preprocessing - converting to grayscale, adjusting contrast, and denoising before OCR processing
  • Large File Handling: Processing large PDFs was causing timeouts. Implemented chunked processing and progress indicators to handle files up to 16MB smoothly
  • Model Loading Time: BART model was taking too long to load. Implemented model caching and warm-up requests to keep it ready
  • Mobile Responsiveness: The drag-drop interface wasn't working well on mobile. Added touch event handlers and fallback file input for better mobile UX

üìä Impact & Results:

Since deploying Info Snap, it has processed over 500+ documents with an average user satisfaction rating of 4.8/5. Users particularly love the translation feature and the ability to share summaries without creating accounts. The app maintains a 98% uptime with average response time under 3 seconds for most documents.
